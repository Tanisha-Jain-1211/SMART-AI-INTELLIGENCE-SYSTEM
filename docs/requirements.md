# Smart Complaint Intelligence System - Requirements

## Citizen User Stories
- As a citizen, I can register and log in securely so that my complaints are linked to my account.
- As a citizen, I can submit a complaint with category, description, image, and location so that authorities can take action.
- As a citizen, I can track complaint status by complaint ID so that I know progress transparently.
- As a citizen, I can receive clear status updates when my complaint moves from pending to resolved.

## Admin User Stories
- As an admin, I can view all complaints across categories so that I can monitor city-wide service quality.
- As an admin, I can assign complaints to departments so that ownership is clear and response is faster.
- As an admin, I can update complaint status and add notes so that field actions are recorded.
- As an admin, I can view analytics dashboards by category, urgency, and status so that I can prioritize resources.

## System (AI) User Stories
- As the AI system, I classify incoming complaints into domain categories to reduce manual triage.
- As the AI system, I predict urgency level to help prioritize high-impact issues.
- As the AI system, I detect potential duplicate complaints to avoid redundant processing.
- As the AI system, I return confidence scores so that admins can assess reliability and override when needed.
